# /plan

`planner` サブエージェントを起動し、計画フェーズを独立コンテキストで実行する。

## 動作

1. 引数として渡された開発タイトル（未指定ならユーザーに確認）を取得する
2. `Agent` ツールで `subagent_type: planner` を起動する
3. 各フェーズ完了後、エージェントが返したサマリをユーザーに提示し、承認を得てから次フェーズへ進む
4. 承認後は再度 `planner` を呼び出し、次フェーズを実行する

詳細な手順・成果物の構成・禁止事項は `planner` エージェントの定義を単一情報源とする（このファイルでは重複させない）。

## 引数

開発タイトル（kebab-case 推奨）。
