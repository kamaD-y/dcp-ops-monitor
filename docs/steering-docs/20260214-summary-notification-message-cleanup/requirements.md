# 要求定義: SummaryNotification 通知メッセージ整理

## 背景

別 PR にてデータ層を S3 から Google Spreadsheet に移行した。
これにより、total（総評価）情報は各 product の情報から算出可能となり、データ保存の対象外となった。
この変更に合わせて、ドメインモデルと通知メッセージの内容を整理する。

## 要求

### 1. DcpAssets モデルから total フィールドを削除

- `DcpAssets` の `total: DcpAssetInfo` フィールドを削除する
- total は `products` の各値を合算して算出する（導出値として扱う）

### 2. 通知メッセージの内容変更

通知メッセージに含める内容を以下に変更する:

#### サマリ（最新データから算出）

| 項目 | 説明 |
|------|------|
| 拠出金額累計 | 全商品合計の拠出金額累計 |
| 評価損益 | 全商品合計の評価損益 |
| 資産評価額 | 全商品合計の資産評価額 |
| 運用年数 | 運用開始からの年数 |
| 運用利回り | 年間運用利回り |
| 想定受取額(60歳) | 60歳時点の想定受取額 |

#### 直近1週間の資産評価額推移

直近カレンダー7日分のデータについて、日毎の資産評価額（全商品合計）と前日比を一覧表示する。

**表示形式:**

```
2026-02-14: 1,200,000円 +5,000円
2026-02-13: 1,195,000円 -2,000円
2026-02-12: 1,197,000円 +3,000円
2026-02-11: 1,194,000円 +1,000円
2026-02-10: 1,193,000円 -
```

- 各行: `日付: 資産評価額（全商品合計） 前日比`
- 前日比は直前のデータポイントとの資産評価額の差分
- 前日データが存在しない場合（期間内の最古のデータ等）は前日比を「-」と表示する
- 該当日にデータが存在しない日（休日等）は行自体を表示しない

### 3. 通知メッセージから削除する項目

- **商品別セクション**: 商品ごとの取得価額累計・損益・資産評価額の一覧
- **目標利回り**: 固定値 0.06 の表示

## 対象外

- 通知スケジュールの変更
- 通知先（LINE）の変更
- 新規データ項目の追加
